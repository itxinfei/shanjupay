闪聚支付-实战-讲义

1 需求概述

1.1 商户资质审核

平台运营人员对商户提交的资质申请信息进行审核，审核通过后才能使用平台提供的聚合支付功能。

1.2 门店管理

使用闪聚支付的商户在不同的地理区域会有多个分店，商户可以通过平台对旗下的门店进行管理：主要包括新增门店、查询门店、修改门店和删除门店等功能。

1.3 员工管理

商户对自家员工进行管理，主要包括新增员工、查询员工列表、修改员工信息和删除员工等功能。

 

1.4 业务流程

1.4.1 商户资质申请

资质申请完整流程如下，其中商户注册和资质申请由商户管理员来完成，资质审核由平台运营人员处理：

具体操作如下：

  1. 商户填写账号、密码、手机号

  2. 填写完成后，点击获取验证码

  3. 输入验证码，点击注册

  4. 商户注册成功，自动跳转到登录页面

  5. 商户填写资质申请信息

  6. 上传营业执照和法人身份证图片

  7. 提交资质申请信息

  8. 平台运营人员登录管理后台，选择企业管理-》审核管理

  9. 打开审核管理列表进行操作

  10. 审核通过后完成资质申请，商户即可进行后续操作

     

1.4.2 门店管理

  1. 商户管理员登录平台后，选择组织管理-》门店管理

  2. 新增门店：点击新建，录入门店信息

  3. 修改门店：点击对应门店后的编辑按钮修改门店信息

  4. 删除门店：点击对应门店后的删除按钮删除门店

1.4.3 员工管理

  1. 商户管理员登录平台后，选择组织管理-》成员管理

  2. 新增员工：点击新建按钮，录入员工信息后提交

  3. 查询员工：输入用户名可进行模糊搜索

  4. 修改员工：点击对应员工后边的编辑按钮

  5. 删除员工：点击对应员工后边的删除按钮

 

2 商户资质审核

需求分析

系统交互流程如下：

  1. 前端请求运营平台应用获取所有待审核的商户列表

  2. 商户平台应用接受后请求商户服务查询列表

  3. 返回待审核商户列表给前端

  4. 管理员点击对应商户后的通过或驳回按钮

  5. 前端向运营平台应用发起资质审核请求

  6. 运营平台应用调用商户服务资质审核接口完成审核

  7. 返回审核结果给前端

3 门店管理

3.1 新增门店

需求分析

系统交互流程如下：

  1. 前端请求商户平台应用获取员工列表

  2. 请求商户服务查询员工列表

  3. 返回员工列表给前端

  4. 管理员填写门店基本信息

  5. 选择门店管理员(可选，可在添加员工设置门店管理员)

  6. 前端提交门店信息到商户平台应用

  7. 调用商户服务新增门店接口

  8. 返回新增结果给前端

    @Override
    public PageVO<StoreDTO> queryStoreByPage(StoreDTO storeDTO, Integer pageNo, Integer pageSize) {
        // 创建分页
        Page<Store> page = new Page<>(pageNo, pageSize);
        // 构造查询条件
        QueryWrapper<Store> qw = new QueryWrapper();
        if (null != storeDTO && null != storeDTO.getMerchantId()) {
            qw.lambda().eq(Store::getMerchantId, storeDTO.getMerchantId());
        }
​
        if (null != storeDTO && null != storeDTO.getStoreName()) {
            qw.lambda().like(Store::getStoreName, storeDTO.getStoreName());
        }
        // 执行查询
        IPage<Store> storeIPage = storeMapper.selectPage(page, qw);
        // entity List转DTO List
        List<StoreDTO> storeList = StoreConvert.INSTANCE.listentity2dto(storeIPage.getRecords());
        // 封装结果集
        return new PageVO<>(storeList, storeIPage.getTotal(), pageNo, pageSize);
    }
